# 迭代规划

## 当前状态

Spicy Chicken Preview — 基于 Tauri v2 + React 19 + TypeScript 的 macOS 图片查看器。定位为「简约但功能齐全」的中度图片浏览器，介于极简查看器（qView、LilyView）与重型管理器（XnView、Bridge）之间。

### 已实现

- 图片查看（Canvas 渲染、Retina 高 DPI、缩放/拖拽、双击全屏）
- 文件夹浏览（侧边栏文件树、子文件夹切换、返回上级）
- 缩略图列表（sips 生成、懒加载、IntersectionObserver）
- 可拖拽调整侧边栏宽度（140px - 500px，持久化）
- 暗色/浅色主题切换（Obsidian / Paper）
- 中英文切换
- 键盘快捷键（← → 切换图片、Space 幻灯片、F 全屏、0 重置缩放、Cmd+/- 缩放、Cmd+B 切换侧边栏）
- 图片预加载（前/后一张图缓存，切换无白屏）
- macOS「打开方式」集成（冷启动 + 热启动）
- 全屏模式（自动隐藏底部缩略图条）
- 幻灯片模式（交叉淡入淡出过渡、可切换间隔 2s/3s/5s/10s）
- 排序功能（名称、尺寸、长宽比、修改时间，升降序）
- 状态栏（图片尺寸、文件大小、格式、缩放比例）
- 拖拽打开（从 Finder 拖拽图片到窗口）
- 最近文件夹（持久化、可展开收起）

### 技术栈

| 层 | 技术 |
|----|------|
| 桌面框架 | Tauri v2 |
| 前端 | React 19 + TypeScript 5.8 + Vite 7 |
| 后端 | Rust (serde, urlencoding, image) |
| 缩略图 | macOS sips CLI |
| 包管理 | pnpm |

## Phase 1 — 基础体验补全

目标：消除日常使用中的阻碍，达到「可以替代 Preview.app」的水平。

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 右键菜单 | 复制图片到剪贴板、在 Finder 中显示、复制文件路径、移动到废纸篓 |
| P0 | EXIF 方向自动纠正 | 读取 EXIF Orientation 标签，自动旋转显示方向，解决竖拍照片横着显示的问题 |
| P0 | 删除/移到废纸篓 | 快捷键 Delete/Backspace，确认后移入废纸篓（macOS Trash） |
| P1 | 图片视觉旋转 | 90° 左转/右转（仅视觉旋转，不修改原文件），快捷键 Cmd+L / Cmd+R |
| P1 | 触控板手势导航 | 双指左右滑动切换图片（类似 LilyView 的核心交互） |
| P1 | 窗口标题栏显示文件名 | 当前仅工具栏显示文件名，窗口标题应同步 |

## Phase 2 — 浏览体验升级

目标：打造同类最佳的文件夹浏览体验。

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 网格视图 | 缩略图网格模式，替代/补充列表视图。可调节缩略图大小（小/中/大）。这是竞品分析中呼声最高的功能 |
| P1 | 面包屑导航 | 顶部路径层级显示，点击任意层级快速跳转，替代纯文字显示 |
| P1 | 文件过滤 | 按格式类型过滤（仅 JPEG、仅 RAW 等），工具栏下拉或侧边栏筛选器 |
| P2 | 收藏夹 | 固定常用文件夹到侧边栏，持久化存储，快速访问 |
| P2 | 多选操作 | 网格视图中多选图片，批量删除/复制/移动 |

---

## Phase 3 — 查看增强

目标：丰富单图查看时的能力和信息展示。

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P1 | 详细元数据面板 | 可切换的侧边面板，显示 EXIF（相机型号、光圈、快门、ISO、焦距、GPS）、文件属性、直方图 |
| P1 | 缩放模式切换 | 在「适应窗口」/「实际大小」/「填充宽度」/「填充高度」之间循环切换 |
| P1 | 鼠标悬停时显示导航箭头 | 图片左右两侧显示半透明箭头按钮，方便纯鼠标操作 |
| P2 | 实际大小快捷查看 | 按住鼠标左键/长按显示 100% 局部（类似 Photo Mechanic 的 Enter 键缩放逻辑） |
| P2 | 对比模式 | 并排显示两张图片，同步缩放和平移，用于比较/选片 |

---

## Phase 4 — 格式与性能

目标：扩展格式支持，优化极端场景性能。

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P1 | HEIC/HEIF 支持优化 | 确保 HEIC 图片正常显示、缩略图生成、EXIF 读取 |
| P1 | AVIF 支持验证 | 测试 AVIF 解码和缩略图生成 |
| P2 | RAW 基础预览 | 利用 macOS 原生 RAW 引擎或 libraw 提供基础 RAW 预览（至少显示嵌入式 JPEG 缩略图） |
| P2 | 大文件夹性能优化 | 10000+ 图片文件夹的浏览流畅度，虚拟滚动、分页加载等 |
| P3 | 动画 GIF/APNG 控制 | 播放/暂停/逐帧查看动画图片 |

---

## Phase 5 — 跨平台

目标：将 macOS 体验带到 Windows 和 Linux，核心功能保持一致。

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P1 | Windows 缩略图 | 使用 PowerShell / WIC (Windows Imaging Component) 替代 sips |
| P1 | Windows 构建与测试 | Tauri Windows 构建配置，基础功能验证 |
| P2 | Linux 缩略图 | 使用 ImageMagick / ffmpegthumbnailer 生成缩略图 |
| P2 | 平台适配 | 各平台文件关联、窗口行为、快捷键差异处理 |

## Phase 6 — 高级功能（远期）

目标：面向高级用户的效率工具，接近 ApolloOne / XnView 的能力。

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P2 | 批量重命名 | 模式匹配、序号、日期替换等 |
| P2 | 批量格式转换 | 格式转换、尺寸调整、质量设置 |
| P3 | 基础编辑 | 裁剪、调整大小、旋转（修改原文件） |
| P3 | 打印 | 基础打印功能，纸张尺寸和边距设置 |

## 建议

下一个迭代建议优先做 **Phase 1 的 P0 项：右键菜单 + EXIF 方向 + 移到废纸篓**。这三个功能是日常使用中最频繁遇到的需求，也是当前与 Preview.app / qView 对比中最明显的短板。
